# References

Cross-cutting Reference-Files für den Skill. Anders als `phases/` (sequenziell)
oder `methods/` (selektierbar) sind diese **kontinuierlich gültig**.

| File | Inhalt | Wann laden |
|------|--------|------------|
| `routing-matrix.md` | Trigger-Phrase → Phase + Skill-Stack Mapping | Mehrdeutige Trigger, Workflow-Pivot |
| `ncp-integration-contract.md` | NCP-Slot-pro-Phase, Mutation-Workflow, Validation-Regeln | NCP-Edit-Frage, Phase 2/3/5 |
| `sc-command-mapping.md` | /sc:-Commands pro Phase mit Use-Cases | /sc:-Aufruf-Frage |
| `significance-heuristics.md` | Checkpoint-Trigger-Detail (übernommen aus Legacy, generalisiert) | Vor Checkpoint-Entscheidung |
| `learnings.md` | Self-Improvement-Log (generisch, projekt-agnostisch) | Bei Bootstrap + Session-End |
| `skill-improvement-todo.md` | Bekannte Skill-Lücken | Skill-Wartung |

**Note:** Projekt-spezifische Files (`canon/`, `canon-meta.md`, `progress.md`,
`open-questions.md`) leben **NICHT mehr im Skill**, sondern im Projekt-Workspace
unter `/home/claude/novel-projects/<slug>/`.
